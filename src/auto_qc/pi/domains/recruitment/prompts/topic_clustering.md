请对以下 AI 质检问题模式进行主题聚类。

每个模式描述了一种 AI 在招聘电话中的发言质量问题。你的目标是将描述**同一类问题**的模式归为同一主题组。

## 聚类规则

同一组的模式必须同时满足以下三个条件：
1. **触发条件相同**：用户在什么情况下触发了这个违规（如"用户说不要/不考虑""用户问薪资/职责""用户说在忙/开会"）
2. **违规表现相同**：AI 具体做了什么违规动作（如"重复播报同一段话""跳过问题强行推进""直接挂断未约定时间"）
3. **检测方法相同**：通过什么方式识别（如"文本相似度对比""意图识别+关键词匹配"）

如果任何一个条件不同，禁止分为同一组。

## 示例

✅ **应该合并的模式：**
- "AI话术机械重复" 和 "核心话术重复播报"
  → 触发条件相同（用户反馈后仍重复），违规表现相同（重复播报），检测方法相同（文本相似度），应该合并

- "用户拒绝后强行推进流程" 和 "用户拒绝后仍纠缠追问"
  → 触发条件相同（用户表达拒绝），违规表现相同（继续推进），应该合并

❌ **不应该合并的模式：**
- "用户拒绝后强行推进" 和 "回避用户提问强行推进"
  → 触发条件不同（拒绝意图 vs 疑问句式），禁止合并

- "话术机械重复" 和 "结束语未根据场景调整"
  → 违规表现不同（重复 vs 模板化），禁止合并

## 输出格式

返回 JSON 数组，不要包含其他文字：

[
  {
    "topic_name": "简短的主题名称，不超过 10 个字",
    "pattern_ids": ["PTN-0001", "PTN-0003"],
    "pattern_names": ["模式原名1", "模式原名2"]
  }
]

要求：
1. 每个模式只能属于一个主题组
2. 所有模式必须被分配到某个组（不能遗漏）
3. 如果一个模式独立成组，也允许（该模式描述的问题与其他模式无重叠）
4. topic_name 要简短准确
